## 🚀 Возможности

- **Единый интерфейс** для шести независимых утилит
- **Стильный консольный вывод** с цветами, ASCII-артом и прогресс-барами
- **Защита мастер-паролем** для доступа к менеджеру паролей
- **Очистка истории** консоли при навигации и выходе
- **Мгновенный запуск** из командной строки

## 📦 Установка

1. Клонируй репозиторий:

   git clone https://github.com/=-=](https://github.com/xijbscnk-art/DEV-Tools
   cd DEV
Установи зависимости:

pip install -r requirements.txt
Запусти главное меню:

python devtools.py

🖥️ Использование
После запуска ты увидишь загрузочный экран с ASCII-логотипом и краткой сводкой о системе. При первом запуске потребуется придумать мастер-пароль — он будет использоваться для шифрования данных в менеджере паролей.

Затем откроется главное меню:

text
══════════════════════════════════════════════════
ГЛАВНОЕ МЕНЮ
[1] 📁 Scaffolder — создать структуру папок
[2] 💻 SysInfo — информация о системе
[3] 🌲 TreeView — дерево папок
[4] 🧹 Clean — очистка мусора
[5] 🔐 Password Manager — генератор и хранитель паролей
[6] 🧪 Test Runner — запустить тесты проекта
[0] 🚪 Выход
Выбери номер нужного инструмента и следуй подсказкам.

🧰 Инструменты
1. 📁 Scaffolder – создание структуры проекта
Создаёт папки и файлы по текстовому описанию.

Пример ввода:

text
my_project
├── src/
│   ├── main.py
│   └── utils/
│       └── helpers.py
├── tests/
│   └── test_main.py
├── requirements.txt
└── README.md
После ввода (пустая строка завершает ввод) структура будет воссоздана на диске.

Особенности:

Поддержка вложенности любой глубины

Автоматическое определение файлов (с точкой) и папок (без точки или с / на конце)

2. 💻 SysInfo – информация о системе
Собирает подробный отчёт об аппаратном и программном обеспечении ПК.

Выводимые данные:

ОС, версия, хостнейм

Процессор (модель, ядра, частота, загрузка)

Оперативная память (всего, использовано, swap)

Видеокарта(ы) (модель, видеопамять, загрузка, температура) – через nvidia-smi или WMI

Диски и разделы (размер, занято, свободно, модель физического диска и тип SSD/HDD)

Материнская плата (производитель, модель, версия BIOS)

Сетевые адаптеры (IP, MAC)

3. 🌲 TreeView – дерево папок
Выводит древовидную структуру выбранной директории.

Параметры:

Путь к папке (по умолчанию текущая)

Максимальная глубина обхода

Игнорирование указанных папок (например, .git, node_modules)

Сохранение результата в файл

Пример вывода:

text
C:\Projects\devtools
├── devtools.py
├── scaffold.py
├── sysinfo.py
├── treeview.py
├── clean.py
├── password_manager.py
├── requirements.txt
└── README.md
4. 🧹 Clean – очистка мусорных файлов
Рекурсивно удаляет временные и кэшированные файлы, создаваемые в процессе разработки.

Что удаляется:

__pycache__, *.pyc, *.pyo

.pytest_cache, .mypy_cache, .ruff_cache

*.log, .DS_Store, Thumbs.db

(опционально) виртуальные окружения .venv/venv

Параметры:

Путь к папке

--dry-run – только показать, что будет удалено

--include-venv – удалять также папки виртуальных окружений

5. 🔐 Password Manager – генератор и хранитель паролей
Безопасно генерирует и хранит пароли в зашифрованном виде.

Возможности:

Генерация паролей с настройкой длины и состава символов

Хранение в зашифрованном файле (AES-256 через Fernet + PBKDF2)

Мастер-пароль для защиты всей базы (вводится при запуске DevTools)

Просмотр сохранённых паролей в виде таблицы

Безопасность:

Ключ шифрования не хранится на диске — вычисляется из мастер-пароля при каждом запуске

Без мастер-пароля данные расшифровать невозможно

6. 🧪 Test Runner – запуск тестов
Запускает все тесты проекта с помощью pytest.

Что тестируется:

scaffold – парсинг и создание структуры

clean – удаление мусорных файлов

password_manager – генерация паролей, шифрование/дешифрование

При отсутствии pytest предлагает установить его автоматически.

🔒 Безопасность
Мастер-пароль запрашивается один раз при запуске DevTools.

Все данные менеджера паролей хранятся в зашифрованном файле ~/.devtools_passwords.enc.

Используется современное шифрование: PBKDF2 (480 000 итераций) + Fernet (AES-128).

Даже при наличии исходного кода и зашифрованного файла восстановить пароли без мастер-пароля невозможно.

При выходе из программы мастер-пароль удаляется из памяти.

🧪 Тестирование
Для запуска тестов выбери пункт 6 в главном меню или выполни вручную:

pytest tests/ -v
Все тесты находятся в папке tests/ и покрывают ключевые функции утилит.

📁 Структура проекта
text
devtools/
├── devtools.py          # точка входа, меню, аутентификация
├── scaffold.py          # создание структуры папок и файлов
├── sysinfo.py           # сбор информации о системе
├── treeview.py          # построение дерева папок
├── clean.py             # очистка мусорных файлов
├── password_manager.py  # генератор и хранитель паролей
├── tests/               # тесты
│   ├── __init__.py
│   ├── test_scaffold.py
│   ├── test_clean.py
│   └── test_password_manager.py
├── requirements.txt     # зависимости
└── README.md            # этот файл
⚙️ Требования
Python 3.12 или новее

Windows (для работы wmi и nvidia-smi; остальные утилиты кроссплатформенны)

Зависимости из requirements.txt:

rich – красивый консольный вывод

pyfiglet – ASCII-арт

psutil – информация о системе

py-cpuinfo – детализация процессора

wmi – доступ к Windows Management Instrumentation

cryptography – шифрование данных

pytest – запуск тестов
